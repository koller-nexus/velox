# Phase 1 Data Model: Help Commands & Additional Useful Commands

**Feature**: `003-cli-help-commands` | **Date**: 2026-07-01

These are in-memory / presentation entities only. The feature adds no persistent
storage; it reads the existing config store (read-only) and reuses existing
measurement/location types.

## Command (registry entry)

The single source of truth for dispatch, the `help` overview, and per-command
help (FR-001, FR-002, FR-007).

| Field | Type | Notes |
|-------|------|-------|
| `Name` | string | Invocation token, e.g. `help`, `version`, `servers`, `ping`, `config`, `consent`. Unique within the registry. |
| `Summary` | string | One-line description shown in the `velox help` overview. |
| `Usage` | string | Detailed help (synopsis, flags, examples) printed by `help <command>` and `<command> --help`, to stdout. |
| `Run` | func(ctx, args) int | Executes the command; returns a process exit code (`0`/`1`/`2`). |

**Rules**:
- Every command the tool accepts MUST be registered (FR-007); the overview is
  generated by iterating the registry, so no command can exist unlisted.
- Lookup is exact-match on `Name`; an unmatched token is a usage error (FR-004).
- `help` and `<command> --help` render `Usage` identically (SC-002).

## ServerListing (output of `servers`, FR-011)

Reuses `locate.Server` + `geo` distance; the command shapes a view of ~5 items.

| Field | Type | Notes |
|-------|------|-------|
| `Servers` | []ServerEntry | The chosen server plus next-nearest, ~5 total, in proximity order. |
| `LocationUsed` | bool | Whether an IP-based location estimate was applied (consent granted + resolved). |

**ServerEntry**

| Field | Type | Notes |
|-------|------|-------|
| `Machine` | string | ndt7 machine FQDN (from `locate.Server.Machine`). |
| `Site` | string | M-Lab site code (`locate.SiteCode`), e.g. `gru01`. |
| `City` | string | From `locate.Server.City` (may be empty). |
| `Country` | string | From `locate.Server.Country` (may be empty). |
| `DistanceKm` | *float64 | Great-circle km when a location estimate is available; `null`/omitted otherwise. |
| `Selected` | bool | `true` for the one server velox would use for a real run. |

**Rules**: consent gate applies before any location lookup (mirrors
`app.go` `selectServer`); when no estimate/coords, fall back to Locate
proximity order with `DistanceKm` unset (same as `geo.SelectNearest`).

## LatencyResult (output of `ping`, FR-013)

| Field | Type | Notes |
|-------|------|-------|
| `Online` | bool | Connectivity succeeded. |
| `LatencyMs` | float64 | Minimum RTT in ms (same definition as a full run). |
| `JitterMs` | float64 | RTT variation in ms over the short sampling window. |
| `Server` | *ServerInfo | The server sampled (reuse `speedtest.ServerInfo` shape: machine/city/country/isFallback). |
| `DistanceKm` | *float64 | Client→server distance when known; `null` otherwise. |
| `DurationMs` | int64 | Wall-clock duration of the sample. |

**Rules**: no download/upload throughput fields are produced or reported;
values come from the ndt7 download subtest's RTT samples (Constitution IV).

## ConfigView (output of `config`, FR-012)

Read-only projection of the existing `config.Config` plus resolved paths.

| Field | Type | Notes |
|-------|------|-------|
| `ConfigPath` | string | `config.Path()` — absolute path to `config.json`. |
| `ConfigDir` | string | Parent directory of `ConfigPath`. |
| `Consent` | ConsentView | Current decision + timestamp. |
| `GeoEndpoint` | string | Effective geolocation endpoint override (empty = default). |
| `FallbackServer` | *FallbackServerView | Present only when configured. |

**ConsentView**: `{ Decision string (unset|granted|denied), DecidedAt *time }`.
**FallbackServerView**: `{ Machine, DownloadURL, UploadURL string }`.

**Rules**: never performs network I/O or a consent prompt (FR-008); a missing or
corrupt config file resolves to defaults (existing `config.Load` behavior),
never an error.

## Relationships & reuse

- `ServerListing`/`LatencyResult` derive from `locate.Server`,
  `geo.Selection`/`geo.HaversineKm`, and `speedtest`/`ndt7` results — no new
  persisted entities.
- `ConfigView` is a read-only view over `config.Config` and `config.Path()`.
- All JSON field names are stable and documented in `contracts/`.
